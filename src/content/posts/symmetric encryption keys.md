Create the group parameters `p` and `g`. This file defines the mathematical group **used by both parties** for the keys.

```bash
openssl dhparam -out dhparams.pem 2048
```

```
DH Parameters: (2048 bit)
	prime:
		00:b8:e0:9e:43:25:72:b1:fc:e3:15:fa:33:5a:f5:
		a7:75:46:23:df:e0:03:38:29:29:c5:48:3a:85:ff:
		...
		29:a0:58:5b:e4:19:88:17:66:f0:ef:a4:54:72:f6:
		34:5b
	generator: 2 (0x2)
```

Generate Party A key pair

```bash
openssl genpkey -paramfile dhparams.pem -out A_private.pem
```

```
DH Private-Key: (2048 bit) # [Group 14]
    private-key: (private value a) Randomly generated by host.
		...
        fc:28:a6:38:f0:a8:8a:91:41:67:ea:bf:05:98:10:
        4c
    public-key: # A = g^a mod p
        4f:f7:86:c6:17:05:04:cb:05:a9:b2:0a:a6:82:af:
		...
        e8:4a:1e:06:9e:9d:14:a3:d1:1d:e8:6a:a8:84:b6:
        ae
    prime: # (p) Public modulus (from dhparams.pem)
		...
        29:a0:58:5b:e4:19:88:17:66:f0:ef:a4:54:72:f6:
        34:5b
    generator: 2 (0x2) (g) (from dhparams.pem)
```

- `OpenSSL` reads the `dhparams.pem` to get `p` and `g`. 
- Generates a random private value `a` for the key
- Computes the corresponding public value `A = g^a mod p`

Extract A's the public key

```bash
openssl pkey -in A_private.pem -pubout -out A_public.pem
```

Generate Party B a key pair and extract public key

```bash
openssl genpkey -paramfile dhparams.pem -out B_private.pem
openssl pkey -in B_private.pem -pubout -out B_public.pem
```

Parties exchange public keys.

Compute the shared secret

Party A

```bash
openssl pkeyutl -derive \
-inkey A_private.pem -peerkey B_public.pem \
-out secret_A.bin
```

Party B

```bash
openssl pkeyutl -derive \
-inkey B_private.pem -peerkey A_public.pem \
-out secret_B.bin
```

Both keys are identical.

```bash
sha256sum secret_A.bin 
#ca2de6138dd91109857ecd31ff8d943cdf1ecb92b7ed7108e83a1917bed21779  secret_A.bin
sha256sum secret_B.bin 
#ca2de6138dd91109857ecd31ff8d943cdf1ecb92b7ed7108e83a1917bed21779  secret_B.bin
```

## Derive Symmetric Keys

The symmetric keys are derived from the shared secret and a chosen algorithm.

Derive a symmetric key from the shared secret. The hash is binary encoded. 

```bash
openssl dgst -sha256 -binary secret_A.bin > sym_key.bin
```

The hashed key is exactly 32-byte, 256-bit long. 2^256 possible combinations.

```
9f 5e 6b f6 ba 86 8f 8e  7d 8c db 7c 16 9f dd ec  |.^k.....}..|....|
bc 0c 47 2f 29 6a 3f a7  8f e5 7c 0e 7e 44 b3 25  |..G/)j?...|.~D.%|
```

The encryption key must be hex.

```bash
KEY=$(xxd -p sym_key.bin | tr -d '\n')
```

Encrypt. Both parties must agree on the key, the initialization vector and the cipher (`AES-256-CBC`)

```bash
openssl enc -aes-256-cbc \
-in message.txt -out message.enc \
-K $KEY \
-iv 00000000000000000000000000000000
```

- `-K`: Hex-encoded key
- `-v`: Initialization vector

De-crypt

```bash
openssl enc -d -aes-256-cbc \
-in message.enc -out message_decrypted.txt \
-K $KEY \
-iv 00000000000000000000000000000000
```

The keys can be derived with a `KDF` (Key derivation function) as well.

#### Elliptic Curves Key Derivation

Elliptic Curve DH uses fixed named elliptic curves instead of modular exponentiation with large primer for key exchange.

List supported curves

```bash
openssl ecparam -list_curves
```

```
secp224r1 : NIST/SECG curve over a 224 bit prime field
secp256k1 : SECG curve over a 256 bit prime field
secp384r1 : NIST/SECG curve over a 384 bit prime field
secp521r1 : NIST/SECG curve over a 521 bit prime field
prime256v1: X9.62/SECG curve over a 256 bit prime field
```

Generate Party A Key and extract the public key

```bash
openssl ecparam -name secp256k1 -genkey -out A_ec_private.pem
```

```bash
openssl pkey -in A_ec_private.pem -text -noout
```

```
Private-Key: (256 bit)
priv: # private secret number d. 
    2d:7a:bd:4f:32:3f:59:9f:90:38:1a:05:06:5e:c0:
    82:f5:f8:92:53:66:4d:12:de:1e:cd:29:f5:d3:0e:
    6b:6a
pub: # Public key point Q on the curve. Q = d.G Shared value
    04:e9:8f:c8:aa:83:7b:15:93:4a:66:91:03:cc:32:
    fa:71:cd:72:1b:46:ce:25:39:0d:a7:73:e1:55:40:
    60:89:c1:a4:7a
ASN1 OID: secp256k1 # name of the curve
```

`d_A`: Private key for Party A
`G`: Generator point of the curve. Base point from which all public keys are derived. Fixed point on the elliptic curve defined by its x, y coordinates. Public.
`Q_B`: Peer public key

`S = d_A.QB`
`S = d_B.QA`

Extract the Party A public key

```bash
openssl pkey -in A_ec_private.pem -pubout -out A_ec_public.pem
```

Generate Party B Key and extract public key

```bash
openssl ecparam -name secp256k1 -genkey -out B_ec_private.pem
openssl pkey -in B_ec_private.pem -pubout -out B_ec_public.pem
```

Compute the shared secret

```bash
openssl pkeyutl -derive \
-inkey A_ec_private.pem -peerkey B_ec_public.pem \
-out ec_secret_A.bin
```

```bash
openssl pkeyutl -derive \
-inkey B_ec_private.pem -peerkey A_ec_public.pem \
-out ec_secret_B.bin
```

Compare both keys

```bash
5fc74eb4b316055cb3c...b3bd808cad8e0c35e05f3ccc9a580  ec_secret_A.bin
5fc74eb4b316055cb3c...b3bd808cad8e0c35e05f3ccc9a580  ec_secret_B.bin
```

Derive a symmetric encryption key

```bash
openssl dgst -sha256 -binary ec_secret_A.bin > ec_key.bin
```

Encrypt using `AES-256-CBC` with the raw key

```bash
KEY=$(xxd -p ec_key.bin | tr -d '\n')
openssl enc -aes-256-cbc -in message.txt \
-out message.enc \
-K $KEY -iv 00000000000000000000000000000000
```

De-crypt

```bash
KEY=$(xxd -p ec_key.bin | tr -d '\n')
openssl enc -aes-256-cbc -in message.enc \
-out message.txt \
-K $KEY -iv 00000000000000000000000000000000
```

#### HMAC keys

Using a shared secret to derive the `HMAC` keys `shared_secret.bin`

Use `HKDF` (Key Derivation Function) to create a hex-encoded  key from a shared secret key and info (label):

```bash
openssl kdf -keylen 16 -kdfopt digest:SHA2-256 \
-kdfopt key:'shared_secret.bin' \
-kdfopt info:'label' HKDF > hmac.key
```

The label separates keys derived from the same secret. 

```
shared_secret
      │
      ├── HKDF(info="client traffic") → client encryption key
      ├── HKDF(info="server traffic") → server encryption key
      └── HKDF(info="exporter")       → exported secret
```

```
13:C2:0B:74:66:69:2D:ED:D0:F6:48:24:F3:76:12:09
```

Generate the MAC tag

```bash
openssl dgst -sha256 -mac HMAC -macopt key:hmac.key plaintext.txt
```

```
HMAC-SHA2-256(plaintext.txt)= 0ff21fe0470a3cb757f4f8594f3f31058fee809833f29bc48008c1ef5cc21b56
```

##### Pre-shared Authentication Key (Deprecated Method)

The key is used to compute an `HMAC` signature over `OpenVPN` control packets.
The receiver verifies the `HMAC` before processing the packet.

This protects from several attacks:
- `DoS`: Packets without a valid `HMAC` are dropped **before TLS processing**
- Port scanning: Without a correct `HMAC` the `OpenVPN` port appears closed to scanners
- Blocks unauthenticated TLS handshake attempts.

#### Subject/Authority Key Identifiers

The key identifiers are the the `SHA1` hashes of the respective public keys.

```
X509v3 Subject Key Identifier: 
    D0:27:07:25:C1:B1:FC:CB:02:75:E2:E7:29:6C:38:4A:E0:F4:A3:EB
```

Raw structure

```
SubjectPublicKeyInfo  
SEQUENCE  
AlgorithmIdentifier  
subjectPublicKey BIT STRING
```

```
  168:d=4  hl=2 l=   7 prim: OBJECT            :id-ecPublicKey
  177:d=4  hl=2 l=   5 prim: OBJECT            :secp384r1
  184:d=3  hl=2 l=  98 prim: BIT STRING        # Public key starts here
  284:d=2  hl=2 l=  66 cons: cont [ 3 ]        
  286:d=3  hl=2 l=  64 cons: SEQUENCE          
  288:d=4  hl=2 l=  29 cons: SEQUENCE          
  290:d=5  hl=2 l=   3 prim: OBJECT        :X509v3 Subject Key Identifier
  295:d=5  hl=2 l=  22 prim: OCTET STRING      [HEX DUMP]:0414D027.F4A3EB
```

Extract the raw public key from the certificate

```bash
openssl asn1parse -in we1.crt -strparse 184 -noout -out we1.pub.raw
```

Calculate the hash

```bash
sha1sum we1.pub.raw
```

```
SHA1(pubkey.raw)= d0270725c1b1fccb0275e2e7296c384ae0f4a3eb
```

The same can be achieved with a separate `PEM` encoded public key

```bash
openssl pkey -pubin -in we1.pub.pem -outform DER > we1.pub.der
```

Inspect the `ASN.1`

```bash
openssl asn1parse -in we1.pub.der -inform DER
```

Extract the `BIT STRING`

```bash
openssl asn1parse -in we1.pub.der -inform DER -strparse 20 -noout -out we1.pub.raw
```
